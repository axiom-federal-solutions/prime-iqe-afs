1. Project Overview

PRIME is a fully autonomous 9-agent AI system built for Walker Contractors LLC to find, score, price, and bid on federal government contracts — with zero manual intervention once running. The platform integrates SAM.gov, Supabase, Anthropic Claude API, and GitHub Actions into a continuous pipeline that operates 24/7.

Core value proposition:

  • Automatically scans SAM.gov for opportunities matching Walker Contractors' NAICS codes
  • AI scores each contract for win probability using the JUDGE agent
  • Prices bids intelligently using VAULT (cost library) + BID ENGINE
  • Drafts proposal documents automatically via the DRAFT agent
  • Tracks wins, losses, and revenue in LEDGER
  • Sends executive alerts and daily digests via EXEC + BRANDI

4. Key Technical Discoveries This Session

4.1 SAM.gov IP Blocking

SAM.gov's Istio gateway blocks GitHub Actions runners (AWS EC2 IPs) with HTTP/2 404 responses in under 1ms — meaning the block happens at the network gateway before the request even reaches the API. Direct fetch calls from GitHub Actions are completely non-functional.

Solution Applied Route all SAM.gov requests through a Supabase Edge Function (sam-proxy) deployed in Deno. Supabase Edge Functions run on Deno Deploy's infrastructure, which has IPs that SAM.gov does not block. The Edge Function accepts the NAICS code, date range, and API key — calls SAM.gov — and returns the raw response wrapped in a JSON envelope: { samStatus, samHeaders, samBody }.

4.2 Two Supabase Accounts — Critical Discovery

The build revealed two separate Supabase accounts were in play, causing workflow failures:

  • MCP / task-dashboard account: Project yazxnngcceygviveiqsg — where sam-proxy was initially deployed (wrong target)
  • prime-db account: Project lsgaifejjoxqudjhkeev (axiom-federal-solutions org) — the CORRECT project for PRIME
Key Rule sam-proxy must be deployed to prime-db (lsgaifejjoxqudjhkeev), and all GitHub secrets must reference that project's URL and service_role key.

4.3 GitHub Actions Secret Mismatch (Run #16 Failure)

After switching SUPABASE_URL to the MCP project to locate sam-proxy, the SUPABASE_SERVICE_KEY still belonged to prime-db — causing JUDGE to fail with "Invalid API key." Both URL and key must point to the same Supabase project.

4.4 CodeMirror 6 In-Browser Code Editing

GitHub's online editor uses CodeMirror 6, requiring programmatic edits via the CM view dispatch API. All in-browser edits used: document.querySelector('.cm-content').cmTile.view and view.dispatch({changes: {from, to, insert}}) for precise character-level replacements.
